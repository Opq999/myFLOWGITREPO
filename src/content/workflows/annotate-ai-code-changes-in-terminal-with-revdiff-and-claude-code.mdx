## What you'll get
You will learn how to use Revdiff, a terminal-based tool, to review and annotate AI-generated code changes and plans without leaving your terminal session. This workflow is particularly useful for iterating on AI-assisted coding tasks, allowing for quick feedback loops directly within your development environment.

## Tools you need
*   **Revdiff** (free) — A terminal UI (TUI) diff viewer designed for reviewing AI-generated code and plans with inline annotations.
*   **Claude Code** (freemium) — An AI coding assistant that runs within your terminal, which Revdiff integrates with for seamless annotation feedback.

## Steps
1.  **Install Revdiff** — Follow the installation instructions provided in the Revdiff GitHub repository to set up the tool on your system. The author does not specify exact installation commands, so refer to the project's README for the most current method.
2.  **Initiate a Review with Claude Code** — Within your Claude Code session, trigger the diff review process. The author provides example commands:

    ```
    /revdiff main
    ```
    
    Alternatively, you can use a natural language prompt like:

    ```
    review diff for last 3 commits
    ```
    
    You should see Revdiff open as a terminal overlay, displaying the code changes.
3.  **Annotate Code Changes** — Read through the diff presented in Revdiff. To add an annotation, you can drop it on any line, hunk (a block of related changes), or file. The author does not detail the exact annotation command, so experiment within the Revdiff interface or consult its documentation for specific instructions.
4.  **Exit Revdiff and Send Annotations** — Once you have finished annotating, quit Revdiff. Your annotations will be automatically sent back to the Claude Code agent.
5.  **Agent Revises Based on Annotations** — Claude Code will receive your annotations and begin working on the suggested changes. Revdiff will then reopen automatically on the new set of changes for your review.
6.  **Review AI-Generated Plans** — For AI-generated plans, use the `revdiff-planning` plugin. When the agent finishes a plan and calls `ExitPlanMode`, Revdiff will automatically open on the plan text.
7.  **Annotate the Plan** — Review the plan presented in Revdiff. Add annotations to any parts you disagree with or want the AI to expand upon.
8.  **Exit Revdiff for Plan Revision** — Quit Revdiff after annotating the plan. The agent will then revise the plan based on your feedback before generating any code.

## Notes & variations
*   **Free Tier Viability**: Both Revdiff and Claude Code are free, making this workflow accessible. Claude Code's free tier is generally sufficient for many tasks.
*   **Common Pitfall**: Ensure Revdiff is correctly installed and configured to integrate with your Claude Code session. Incorrect setup might prevent annotations from being sent back to the agent.
*   **Tip for Better Results**: Clearly articulate your feedback in the annotations. Instead of just saying "change this," explain *why* it needs to change or *what* you expect the change to achieve for more effective AI revisions.
