## What you'll get
You'll learn to structure your AI-assisted coding workflows using agentic engineering principles with Claude. This involves defining subagents, commands, and skills to create more reliable and reproducible code. This approach moves beyond simple 'vibe coding' to a more systematic method, improving code quality, consistency, and maintainability, especially for complex projects.

## Tools you need
*   **Claude** (freemium): Your AI assistant for generating and refining code.
*   **GitHub** (free / freemium), For version control, hosting your code, and potentially automating workflows.
*   **Web Browser (e.g., Chrome)** (free): To access Claude's web interface and GitHub.
*   **Slack** (freemium): (Optional) For integrating Claude into team communication for code reviews or notifications.

## Steps
1.  **Understand Agentic Engineering Concepts**: Familiarize yourself with the core components of agentic engineering as applied to coding:
    *   **Subagents**: Specialized AI entities responsible for specific tasks (e.g., a 'Python Function Generator' subagent).
    *   **Skills**: Reusable capabilities or actions that subagents can perform (e.g., a 'code generation' skill).
    *   **Commands**: Instructions that orchestrate subagents and skills to achieve a larger goal (e.g., a 'generate feature' command).
    You should have a basic understanding of how these components work together to structure an AI workflow.

2.  **Set up your project for agentic workflows**: Create a dedicated directory structure within your coding project to house your agentic definitions. This helps keep your AI configurations organized and version-controlled.
    *   **What to do**: In your project's root directory, create a new folder named `.claude`. Inside `.claude`, create three subfolders: `agents`, `skills`, and `commands`.
    *   **What you should see**: Your project now has a `.claude` directory containing `agents/`, `skills/`, and `commands/` subdirectories.

3.  **Define a Subagent's Role**: Use Claude to help you draft the definition for a specific subagent. This definition will outline its purpose, responsibilities, and how it interacts with other components.

## Notes & variations
*   **Free-tier alternatives**: Claude offers a free tier that is sufficient for experimenting with these concepts. GitHub is free for public repositories, making it ideal for hosting your project and its `.claude` definitions.
*   **Common mistake**: A common pitfall is treating Claude like a simple chatbot for every task. Agentic engineering requires you to define clear roles, inputs, and outputs for your agents and skills. Without this structure, you'll likely get inconsistent or less reliable results.
*   **Tip for better results**: Start small. Define one agent and one skill for a very specific, manageable task. Iterate on these definitions, refining their responsibilities and interactions, before attempting to build more complex orchestration commands. Always use version control (like Git) for your `.claude` definitions, just as you would for your code.
