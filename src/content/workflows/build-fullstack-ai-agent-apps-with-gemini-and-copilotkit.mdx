## What you'll get
You will learn how to build two practical AI agents: a Post Generator for social media content and a Stack Analyzer for inspecting GitHub repositories. This workflow demonstrates a full-stack architecture using modern AI development tools, suitable for creating sophisticated AI applications.

## Tools you need
*   **Google Gemini** (freemium) - A powerful large language model for reasoning and text generation.
*   **CopilotKit** (freemium) - An SDK to easily embed AI agents into your frontend UI.
*   **LangGraph** (free) - A library for building stateful agent workflows.
*   **Next.js** (free) - A React framework for building the frontend and API routes.
*   **FastAPI** (free) - A modern, fast web framework for building the Python backend.

## Steps
1.  **Understand the Architecture** - Familiarize yourself with the project structure, which separates the Next.js frontend (including UI and API routes) from the Python/FastAPI backend hosting the LangGraph agents. The frontend uses CopilotKit to interact with the backend agents.
2.  **Set up the Frontend Project** - Initialize a Next.js project. Ensure you have the necessary CopilotKit packages installed (`@copilotkit/react-core`, `@copilotkit/runtime`, `@copilotkit/react-ui`). The `layout.tsx` file will wrap your application with the `CopilotKit` provider.
3.  **Implement the Post Generator UI** - Create a new route in your Next.js app (e.g., `frontend/app/post-generator/page.tsx`). This UI will allow users to input prompts for content generation. It will use CopilotKit's chat components to display the conversation and agent responses.
4.  **Develop the Post Generator Backend** - In the `agent/` directory, create `posts_generator_agent.py`. This Python file will define the LangGraph workflow for the post generator. It will involve nodes for gathering context (e.g., via web search) and generating content using Google Gemini.
5.  **Integrate Post Generator with FastAPI** - In `agent/main.py`, register the post generator agent and expose it as an API endpoint using FastAPI. This endpoint will be called by the Next.js API route (`frontend/app/api/copilotkit`).
6.  **Set up the Stack Analyzer UI** - Create another route in your Next.js app (e.g., `frontend/app/stack-analyzer/page.tsx`). This UI will accept a GitHub repository URL as input and display the analysis results.
7.  **Develop the Stack Analyzer Backend** - In `agent/stack_agent.py`, define the LangGraph workflow for the stack analyzer. This workflow will include nodes to gather context from the GitHub repo (metadata, README, code files) and then analyze this information using Google Gemini to infer the tech stack.
8.  **Integrate Stack Analyzer with FastAPI** - Register the stack analyzer agent in `agent/main.py` and expose it via a FastAPI endpoint, similar to the post generator.
9.  **Connect Frontend API Routes** - In your Next.js API routes (e.g., `frontend/app/api/copilotkit`), write the logic to forward requests from the UI to the appropriate FastAPI backend endpoints.
10. **Test the Agents** - Run both the Next.js development server and the FastAPI backend. Test the Post Generator by providing a topic and seeing if it drafts content. Test the Stack Analyzer by providing a GitHub URL and verifying the inferred tech stack.

## Notes & variations
*   **Free Tier Tip:** Ensure you are using the free tiers of Google Gemini and CopilotKit. For local development, you can run FastAPI and Next.js applications without cost.
*   **Common Pitfall:** Incorrectly configuring the API routes between Next.js and FastAPI can lead to connection errors. Double-check the endpoint URLs and request/response formats.
*   **Better Results:** For more accurate stack analysis, consider adding more sophisticated context gathering steps in the LangGraph workflow, such as parsing specific configuration files (e.g., `package.json`, `requirements.txt`) directly.
