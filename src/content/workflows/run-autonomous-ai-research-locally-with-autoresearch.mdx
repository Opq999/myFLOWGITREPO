## What you'll get
You will set up and run a local, autonomous AI research experiment on your own machine using CPU. This approach bypasses complex frameworks, focusing on a tight feedback loop for efficient, self-contained AI development.

## Tools you need
*   **autoresearch** (free) — An open-source repository for running autonomous AI research experiments.
*   **Codex 5.3** (paid) — A coding assistant used here for planning, checking dependencies, and refining the execution environment.
*   **GitHub** (free tier available) — For forking and cloning the autoresearch repository.
*   **A code editor** (e.g., VS Code, free) — To open and work with the cloned repository files.

## Steps
1.  **Fork and clone the repository** — Go to the autoresearch GitHub repository and fork it to your own account. Then, clone your forked repository to your local machine.
    You should have a local copy of the autoresearch project files on your computer.
2.  **Open the project in your coding assistant** — Load the cloned repository into your preferred code editor. The author used Codex 5.3 for its planning and dependency checking capabilities.
    Your coding environment should now be set up to analyze and modify the project files.
3.  **Prepare the experiment environment** — Use your coding assistant to verify that all necessary dependencies are installed and that the prerequisite training steps are ready. This ensures your local setup is correct.
    You should see confirmation that dependencies are met and the environment is prepared for training.
4.  **Define the experiment in `program.md`** — Create a new, standalone experiment definition file. Combine this with the base repository rules to preserve essential constraints from the original setup. The author's experiment focused on whether a structured self-reflection step improves an agent's model improvement decisions.
    You will have a `program.md` file that clearly outlines your specific research experiment.
5.  **Optimize the execution environment** — With your experiment defined, ask your coding assistant (e.g., Codex 5.3) to help refine the setup. The goal is to optimize the environment for running the experiment without altering the core experiment design itself.
    The execution environment should be configured for efficient local runs.
6.  **Run the autonomous research loop** — The autoresearch tool will now read your `program.md` instructions, modify the `train.py` file as needed, run short training experiments, evaluate the results, and decide on the next steps, all within your local machine using CPU.
    You should observe the agent iterating through its defined research loop, making changes, and reporting metrics.

## Notes & variations
*   **Free-tier alternative for coding assistant**: While the author used Codex 5.3, consider using free tools like GitHub Copilot (which has a free tier for students and verified educators) or other open-source code completion tools for planning and dependency checks. However, their reasoning depth might differ.
*   **Common pitfall**: Ensure you have Python and the necessary libraries installed *before* attempting to run the experiment. Dependency issues are a common reason for setup failures.
*   **Tip for better results**: Clearly define your experiment's objective in `program.md`. The more specific your instructions and desired outcomes, the more focused and effective the autonomous agent's research loop will be.
