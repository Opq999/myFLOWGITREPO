## What you'll get
You will create structured HTML reports from your screen recordings. These reports will include keyframes, GIFs of dynamic actions, and an action checklist, making agent feedback clear and actionable. This approach is useful for documenting processes and providing visual, step-by-step guidance.

## Tools you need
*   **Droid** (freemium): A tool that helps convert video feedback into visual reports with keyframes and GIFs.
*   **ffmpeg** (free): A command-line tool for handling multimedia files, used here to potentially create video clips from keyframes.

## Steps
1.  **Record your screen**: Use Droid to record your screen while demonstrating a process or providing feedback. The author mentions using Droid for about 30 minutes for a recording.
    You should have a screen recording file ready for processing.
2.  **Process the recording with Droid**: Input your screen recording into Droid. The tool will automatically process the video to pull out key information.
    Droid will generate a visual report that includes keyframes, GIFs for dynamic parts of the recording, and an 'actions' checklist.
3.  **Review the generated HTML report**: Open the HTML file produced by Droid. This report should visually represent your screen recording with highlighted moments and actions.
    You should see a structured document that is easy to navigate and understand, serving as a build log or feedback document.
4.  **Consider using ffmpeg for clips (Optional)**: If you need actual video clips instead of just GIFs or keyframes, you can use ffmpeg. The author notes this is more for personal preference than necessity for agent understanding.
    The author doesn't provide specific ffmpeg commands, but a general approach would involve identifying timestamps from Droid's output and using ffmpeg to extract segments of the original video.

## Notes & variations
*   **Free tier viability**: Droid offers a freemium model, and its free tier should be sufficient for generating these reports. ffmpeg is completely free.
*   **Common mistake**: Relying too heavily on the tool to perfectly capture every nuance without reviewing the output. Always check the generated HTML report to ensure accuracy and clarity.
*   **Tip for better results**: Experiment with different screen recording lengths and complexity. Shorter, focused recordings often yield clearer and more actionable reports for agents. You can also use Droid's features to highlight specific points on screen during recording for better emphasis. The author suggests that agents read frames well, so focusing on clear visual cues within the recording is key.
