## What you'll get
You'll learn to use AI subagents to parallelize software development tasks, from planning to implementation. This includes generating detailed project tickets and scaffolding new features concurrently.
This approach breaks down complex tasks into smaller, independent parts, allowing specialist AI agents to work on them simultaneously, significantly speeding up development time.

## Tools you need
*   **Claude Code** (paid): An AI development environment or API for orchestrating specialist AI agents to generate code, plans, and documentation.
*   **Linear** (freemium): A project management tool used to create and track development tickets.

## Steps

1.  **Identify a development task**: Pinpoint a new feature or fix needed in your project, such as a missing page or an API integration.

    You should see: A clear idea of the development task you want to address.

2.  **Generate a project ticket with planning agents**: Use your Claude Code environment to run a custom command that invokes planning sub-agents. These agents will collaborate to define requirements, UI, and API needs for your task. The author doesn't share their exact command; a starting point might look like this:

    ```
    /generate-ticket "Create 'View All Insights' page to show all AI-generated analyses for a pull request."
    ```

    You should see: A fully-formed development ticket, potentially created directly in Linear, detailing the requirements for the new feature.

3.  **Review the generated ticket**: Examine the ticket generated by the planning agents. Check if the requirements, UI considerations, and API needs are accurately captured and complete.

    You should see: A clear understanding of the proposed plan. You can make manual adjustments if needed.

4.  **Kick off implementation with coding agents**: Once the plan is defined in the ticket, use another custom command in Claude Code to initiate the implementation phase. This command will feed the ticket details to specialist coding agents (e.g., `senior-software-engineer`, `code-reviewer`) that will work in parallel to build the backend, frontend, tests, and documentation. The author doesn't share their exact command; a starting point might be:

    ```
    /implement-feature --ticket-id [ID of the Linear ticket]
    ```

    You should see: The agents actively working in the background, potentially across multiple terminals, generating code, tests, and documentation for the feature. The author notes they "started hitting API rate limits."

5.  **Monitor and iterate**: Keep an eye on the progress of the implementation agents. If the initial output isn't quite right, you can fire off another run with refined instructions or adjustments to the ticket.

    You should see: Code artifacts (e.g., backend endpoints, UI components, test suites, documentation) being generated and refined. The author emphasizes the low cost of failure, encouraging multiple runs.

6.  **Integrate and test the generated code**: Once the agents have completed their work, integrate the generated code into your project. Run your application and tests to verify the new feature.

    You should see: The new feature working as expected, or clear indications of areas that need further human refinement.

## Notes & variations
*   **Free-tier alternatives**: The core tool, Claude Code, is a paid service. For similar agentic workflows, you might explore open-source agent frameworks like AutoGen or CrewAI, which can be run locally or on free-tier cloud services, though they require significant setup and coding knowledge. For simpler code generation, freemium chat apps like ChatGPT, Claude.ai, or Gemini can assist with individual code snippets, but lack the orchestration capabilities.
*   **Common mistake**: Expecting perfect code on the first try. The author explicitly states, "If the agents get it wrong, I don't really care—I'll just fire off another run." Treat agent output as a starting point, not a final solution, and be prepared to iterate.
*   **Tip for better results**: Clearly define the task and provide as much context as possible in the initial prompt or ticket. For API integrations, providing the full API documentation to the orchestrating agent, as mentioned in the Stripe example, is crucial for accurate and complete code generation.
