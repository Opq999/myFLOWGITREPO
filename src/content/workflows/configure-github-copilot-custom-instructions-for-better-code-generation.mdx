## What you'll get
You will learn how to set up custom instructions for GitHub Copilot to ensure it generates code that aligns with your project's specific patterns and standards. This approach helps prevent Copilot from adopting outdated or undesirable coding practices found in your codebase, leading to more consistent and higher-quality code generation.

## Tools you need
*   **GitHub Copilot** (freemium): An AI pair programmer that suggests code and entire functions in real-time, right inside your editor.
*   **ChatGPT** (freemium): A versatile AI chatbot that can help draft and refine your Copilot instructions.
*   **Leonardo** (freemium): An AI image generation tool, mentioned by the author for potential future use in creating custom banners or visuals.

## Steps
1.  **Understand the Goal of Repo Instructions**: Recognize that just like a new senior developer needs guidance, GitHub Copilot benefits from clear instructions about your project's conventions, architecture, and potential pitfalls. Without them, it might default to your repo's worst patterns.
2.  **Plan for Instruction Setup**: Allocate about 15 minutes to write, test, and refine your custom instructions. This proactive step is crucial for effective AI collaboration.
3.  **Draft Initial Instructions (using ChatGPT)**: The author doesn't share their exact prompt for drafting instructions, but a starting point could be: 
    ```
    I need to write custom instructions for GitHub Copilot for a [describe your project type, e.g., Python web application using Flask]. The goal is to guide Copilot to follow specific coding standards, avoid certain legacy patterns, and adhere to our project's architecture. Please help me draft a set of clear, concise instructions that cover:
    - Preferred coding style (e.g., PEP 8 for Python)
    - Key architectural patterns to follow
    - Any specific libraries or frameworks to prioritize or avoid
    - Common pitfalls or anti-patterns in our existing codebase that Copilot should steer clear of
    - Examples of desired code snippets or structures
    ```
    You should receive a draft of instructions that you can then refine.
4.  **Refine Instructions Based on Project Specifics**: Review the drafted instructions. Add details about your project's unique requirements, such as specific naming conventions, error handling strategies, or preferred ways to interact with your database. The author emphasizes that the goal is to keep Copilot "coloring inside your lines."
5.  **Implement Instructions in GitHub Copilot**: Navigate to your GitHub Copilot settings within your IDE (e.g., VS Code). Locate the section for custom instructions or configuration and paste your refined instructions. The exact location may vary slightly depending on your IDE and Copilot version.
    You should see a text area where you can input your custom instructions.
6.  **Test and Iterate**: After implementing the instructions, start coding as you normally would. Observe Copilot's suggestions. Do they align with your instructions? If not, revisit your instructions, tweak the wording, add more specific examples, or clarify any ambiguities. The author stresses a "build them, repeat" cycle.
    You should see Copilot's suggestions becoming more aligned with your project's needs over time.

## Notes & variations
*   **Free Tier Viability:** GitHub Copilot itself is a paid tool after a trial period. However, the process of drafting and refining instructions can be done using free tools like ChatGPT. For users in Nigeria, consider the cost of Copilot if you plan to use it long-term.
*   **Common Pitfall:** A common mistake is writing instructions that are too vague or too long. Copilot may struggle to interpret overly complex or ambiguous directives. Be specific and focus on the most critical aspects of your project's coding standards.
*   **Tip for Better Results:** Provide concrete examples of code within your instructions. Showing Copilot exactly what you want (e.g., a specific function signature, a class structure, or an error message format) is often more effective than just describing it.
