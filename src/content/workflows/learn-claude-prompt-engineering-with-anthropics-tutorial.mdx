## What you'll get

A solid, hands-on grounding in "prompt engineering", the skill of writing instructions that get good, reliable answers out of an AI. You work through Anthropic's free interactive tutorial: each lesson teaches one idea, you try it in an exercise, and you check your answer. By the end you can structure a clear prompt, spot why a weak prompt failed, and fix it. It is beginner-friendly and the single best way to stop guessing at prompts.

## Tools you need

- **Claude** (freemium): the AI you practice on. The tutorial uses Claude 3 Haiku (the smallest, fastest model), but any Claude model works for learning the ideas.
- **Claude for Sheets** (freemium, optional): a Google Sheets add-on the tutorial can run in, which many find the easiest way to do the exercises.

## Steps

1. **Open the tutorial**: go to Anthropic's Prompt Engineering Interactive Tutorial. The author suggests the "Claude for Sheets" version for the friendliest experience, but the same content is available directly too.

2. **Start at Chapter 1**: begin with "01: Basic Prompt Structure". Do not skip ahead; each chapter builds on the last.

3. **Do every lesson and exercise**: read the lesson, then try the exercise. Use the "Example Playground" to tweak a prompt and watch how Claude's answer changes, that experimenting is where the learning sticks.

4. **Check your answers**: each exercise has an answer key. Compare yours, and re-read the lesson for anything you missed.

5. **Work through the rest in order**: continue chapter by chapter, applying each new technique, until you can build and refine strong prompts on your own.

## Notes & variations

- **Any model works to learn**: the tutorial uses the small, fast Claude 3 Haiku to keep things cheap, but the techniques transfer to any Claude model (and largely to other chatbots too).
- **Sheets vs direct**: the Claude for Sheets version is the gentlest way in; if you prefer, the same lessons run directly without Sheets.
- **Tip for better results**: actually do the exercises rather than just reading. Prompt engineering is a practical skill; tweaking prompts and seeing the effect is how it clicks.
