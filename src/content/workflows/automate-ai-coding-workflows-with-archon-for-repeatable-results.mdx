## What you'll get
You'll learn how to define and run AI coding workflows using Archon, an open-source tool. This allows you to automate tasks like planning, implementation, testing, and code review with AI agents, ensuring consistent and repeatable results every time. This approach eliminates the 'model's mood' problem, making AI coding deterministic by structuring the development process.

## Tools you need
*   **Archon** (free) — A workflow engine for AI coding agents to define and run development processes.
*   **AI Agent (e.g., Claude)** (freemium) — An artificial intelligence model that performs coding tasks within the Archon workflow.
*   **Git** (free) — A version control system essential for managing code and Archon's isolated worktrees.
*   **Bash** (free) — A command-line shell used for executing scripts and deterministic steps in the workflow.
*   **Bun** (free) — A fast JavaScript runtime, package manager, and bundler, used here for running validation scripts.

    *   *What you should see*: Archon can connect to your AI agent.

3.  **Create your project structure** — Archon workflows are defined within your project's `.archon/workflows/` directory.
    *   *What to do*: In your project's root directory, create a folder named `.archon`, and inside it, another folder named `workflows`.
    *   *What you should see*: You have a directory structure like `your-project/.archon/workflows/`.

4.  **Define your AI coding workflow** — Create a YAML file inside the `workflows` directory to define the sequence of AI and deterministic steps. The excerpt provides an example for building a feature.
    *   *What to do*: Create a file named `build-feature.yaml` inside `.archon/workflows/` and paste the following content:

    ```yaml
    nodes:
      - id: plan
        prompt: "Explore the codebase and create an implementation plan"
      - id: implement
        depends_on: [plan]
        loop:
          prompt: "Read the plan. Implement the next task. Run validation."
          until: ALL_TASKS_COMPLETE
          fresh_context: true
      - id: run-tests
        depends_on: [implement]
        bash: "bun run validate"
      - id: review
        depends_on: [run-tests]
        # The excerpt truncates here, but implies a review step. A common prompt might be:
        prompt: "Review the implemented code and tests. Provide feedback and suggest improvements."
    ```

    *   *What you should see*: Your `build-feature.yaml` file is saved, outlining the steps for your AI agent.

5.  **Run the workflow** — Execute your defined Archon workflow from the command line.
    *   *What to do*: Open your terminal or command prompt, navigate to your project's root directory, and run the Archon command to start the workflow. The exact command is not provided in the excerpt, but typically it would be `archon run <workflow-name>`.
    *   *Example*:

    ```
    archon run build-feature
    ```

    *   *What you should see*: Archon starts executing the workflow, showing progress messages as the AI agent performs planning, implementation, testing, and review. You might see output from the `bun run validate` step.

6.  **Review the output and artifacts** — Archon aims to produce a finished PR with review comments.
    *   *What to do*: Once the workflow completes, check your project's Git history or the output from Archon for the generated code, test results, and any pull request descriptions or review comments.

## Notes & variations
*   **Free-tier alternatives**: Archon itself is free and open-source. For AI agents, besides Claude, Google's Gemini and OpenAI's ChatGPT also offer free tiers or low-cost options that can be integrated with Archon.
*   **Common mistake**: A frequent pitfall is not correctly configuring the AI agent's API key or ensuring all necessary dependencies (like `bun` if your workflow uses it) are installed. Always double-check your environment setup and Archon's documentation for specific requirements.
*   **Tip for better results**: Start with simple workflows to understand Archon's mechanics. Gradually add more complex steps and AI interactions. The `fresh_context: true` setting in loops, as shown in the example, is crucial for preventing the AI from getting confused by long conversation histories, ensuring each iteration starts with a clear focus.
