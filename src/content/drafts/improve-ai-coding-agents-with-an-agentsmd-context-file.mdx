## What this is, in plain English
`AGENTS.md` is a special Markdown file that you place in your software project's main folder. It acts like a detailed instruction manual specifically for AI coding agents, telling them how to understand, build, test, and make changes to your code.

Think of it as a `README` file, but instead of being written for a human developer, it's designed for an AI. It provides the exact commands, rules, and important details an AI agent needs to work effectively on your project without constantly asking questions or making mistakes.

This is an advanced concept because it requires you to be comfortable with developer tools, understanding codebases, and potentially using command-line interfaces (CLIs). There isn't a single, simple "copy-paste" recipe, as the exact steps for using `AGENTS.md` will depend on the specific AI coding agent you choose and the unique setup of your software project.

## What you can use it for
*   **Guide AI agents on project setup**: Tell the AI the exact commands to install dependencies (the extra software your project needs) and build the project, so it can get started quickly.
*   **Ensure code quality with tests**: Instruct the AI on how to run tests and linting (tools that check for code errors and style issues), making sure its changes don't break existing code.
*   **Define project structure**: Give the AI a quick map of important files and folders, so it knows where to look for specific code and where to make changes.
*   **Enforce coding conventions**: Specify rules for how code should be written, like naming styles, how to handle errors, or which libraries (collections of pre-written code) to use.
*   **Prevent costly mistakes**: Set clear "guardrails" (specific rules or boundaries) to tell the AI what *not* to do, like editing automatically generated files or committing changes directly to the main version of your code.

## How it actually works
To use `AGENTS.md`, you will integrate it into your existing software development workflow with an AI coding agent. The exact steps will vary based on the specific agent you choose, but the general process involves:

1.  **Choose an AI coding agent**: Select a tool that supports `AGENTS.md` or can benefit from clear project context, such as Cursor, Aider, or an IDE (Integrated Development Environment) with AI features.
2.  **Create the `AGENTS.md` file**: In the main folder of your software project (often called the "root" directory), create a new file named `AGENTS.md`.
3.  **Add project context**: Fill the `AGENTS.md` file with specific, actionable information about your project. The author suggests including:
    *   A single sentence describing what the project is and what technologies it uses (e.g., "A TypeScript REST API on Node 20 with Postgres, deployed to Fly.io.").
    *   Setup and build commands, which are the real, copy-pasteable commands needed to get the project running (e.g., `npm install`, `npm run build`).
    *   Commands to run tests, linting, or type checks (e.g., `npm test`, `npm run lint`).
    *   A short, focused map of key directories and entry points (important files).
    *   Specific coding conventions you want followed (e.g., "Validate all input with Zod at the route boundary").
    *   "Guardrails" or rules about what *not* to do (e.g., "don't edit files in /generated", "never run migrations against production config").
4.  **Use the AI agent**: Interact with your chosen AI coding agent within your project. The agent will automatically read and use the `AGENTS.md` file to understand your project's structure, rules, and commands, helping it to make more accurate and helpful suggestions or changes. The specific way you interact with the agent (e.g., typing commands in a terminal, using an IDE chat window) depends on the tool.

## Words you'll see, explained
*   **AI Coding Agent**: A software tool that uses artificial intelligence to help developers write, debug, and manage code.
*   **Repository (repo)**: A central place where code and other project files are stored, often managed with version control systems like Git.
*   **Markdown file**: A plain text file that uses simple formatting symbols (like `##` for headings or `*` for lists) to create structured documents.
*   **CLI (Command-Line Interface)**: A way to interact with a computer program by typing text commands, instead of clicking buttons in a graphical window.
*   **Guardrails**: Specific rules or boundaries set to prevent an AI from performing certain actions or making mistakes.
*   **Open Standard**: A specification or format that is publicly available and can be used by anyone, promoting compatibility across different tools.

## Notes & variations
*   **Do you even need this?**: For very simple coding tasks or small, one-off scripts, creating a full `AGENTS.md` file might be more effort than it's worth. In such cases, you can often give direct, detailed instructions to an AI chatbot. This file is most useful for larger, more complex software projects where consistency, specific rules, and preventing errors are critical.
*   **Free-tier limits**: Most advanced AI coding agents, especially those integrated into professional development environments, are paid tools. However, open-source options like Aider are free to use, and the Gemini CLI is free to install, though using the Gemini API itself might incur costs depending on your usage.
*   **Common pitfall**: An outdated `AGENTS.md` file is worse than no file at all. If the commands, paths, or rules in the file are incorrect or no longer apply to your project, the AI will confidently follow the wrong instructions, potentially leading to errors or wasted effort. Always keep your `AGENTS.md` file current with your project's state.
*   **Tip for better results**: Keep your `AGENTS.md` file concise and focused. The author notes that every non-essential line dilutes the signal, making it harder for the AI to find and prioritize the most important instructions. Include only what the AI needs to act effectively, not a full project history or human-oriented details.
