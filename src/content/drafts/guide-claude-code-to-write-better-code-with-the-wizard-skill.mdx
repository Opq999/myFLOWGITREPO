## What you'll get
You'll get higher quality, more robust code generated by Claude Code, with fewer bugs and regressions. This approach transforms Claude from a fast but sometimes error-prone coder into a methodical software architect by giving it a structured, multi-phase thinking process before it writes any code.

## Tools you need
*   **Claude Code** (freemium) — Your AI coding assistant, accessible via its CLI or web interface.
*   **GitHub** (freemium) — For project management (issues, pull requests) and storing your project's code and the `/wizard` skill file.
*   **GitHub Actions** (freemium) — For continuous integration (CI) to automate testing and deployment. The `/wizard` skill needs a CI system to respond to in its final phase (Phase 8).

## Steps
1.  **Prepare your GitHub repository** — If you don't already have one, create a new repository on GitHub for your project. This is where your code, `CLAUDE.md`, and the `/wizard` skill file will live.

    You should see an empty or initialized repository ready for your project files.

2.  **Define project conventions in `CLAUDE.md`** — Create a new file named `CLAUDE.md` in the root of your GitHub repository. This file will contain your project's coding standards, architectural guidelines, and any specific conventions Claude should follow.

    *Example content for `CLAUDE.md`:*
    ```markdown
    # Project Conventions for Claude Code

    ## Coding Standards
    *   Use Python 3.9+
    *   Follow PEP 8 style guide.
    *   All functions and classes must have docstrings.
    *   Prioritize readability and maintainability.

    ## Architectural Guidelines
    *   Prefer a microservices architecture where appropriate.
    *   Use FastAPI for new API endpoints.
    *   Database interactions should use SQLAlchemy ORM.

    ## Testing
    *   All new features require 100% test coverage.
    *   Use `pytest` for unit and integration tests.
    *   Practice Test-Driven Development (TDD).

    ## Security
    *   Sanitize all user inputs.
    *   Avoid hardcoding sensitive information.
    *   Implement proper authentication and authorization.
    ```

    You should see the `CLAUDE.md` file committed to your repository, clearly outlining your project's rules.

3.  **Create the `/wizard` skill file** — In your GitHub repository, create a new markdown file. The author implies it's named `/wizard` and lives in the project, so a good practice is to name it `wizard.md` and place it in the project root or a dedicated `.claude` directory. This file will contain the structured methodology Claude will follow.

    You should see an empty `wizard.md` file ready for content.

4.  **Add the Phase 1 methodology to `wizard.md`** — Paste the following content into your `wizard.md` file. This defines the first phase of the author's 8-phase methodology, focusing on planning and assessment. The author describes an 8-phase methodology, but only Phase 1 is detailed in the provided excerpt. The content below is a reconstruction of Phase 1 based on the description. For a complete `/wizard` skill, you would need to define the remaining 7 phases yourself.

    ```markdown
    # /wizard: Methodical Software Architecture Assistant

    This skill guides Claude Code through an 8-phase methodology to ensure high-quality, well-tested code.

    ## Phase 1: Plan Before You Code

    **Objective:** Understand the task, assess complexity, and create a detailed plan before writing any code.

    **Instructions for Claude:**
    1.  **Read Project Conventions:** Carefully review the `CLAUDE.md` file in the project root to understand all project-specific conventions, coding standards, and architectural guidelines.
    2.  **Understand the Task:** Access and thoroughly read the linked GitHub Issue. Identify the core problem, desired outcome, and any specific requirements or constraints.
    3.  **Assess Complexity:**
        *   Identify potentially affected files and modules.
        *   Evaluate architectural impact.
        *   Anticipate potential risks, edge cases, and areas where things could go wrong (e.g., race conditions, data integrity issues, error handling).
    4.  **Size the Work:** Based on the assessment, estimate the scope and effort required.
    5.  **Propose a Structured To-Do List:** Generate a detailed, step-by-step plan for implementing the feature or fix. This plan should include:
        *   High-level steps.
        *   Specific sub-tasks.
        *   Considerations for testing (TDD approach).
        *   Potential refactoring needs.
        *   Any necessary research or design decisions.

    **Output:** Present the structured to-do list, a summary of the complexity assessment, and any clarifying questions before proceeding to the next phase.

    ---
    *(Note: The original author describes an 8-phase methodology, but only Phase 1 is detailed in the provided excerpt. The content above is a reconstruction of Phase 1 based on the description. For a complete /wizard` skill, you would need to define the remaining 7 phases.)*
    ```

    You should see the `wizard.md` file committed to your repository, containing the detailed instructions for Claude's planning phase.

5.  **Create a GitHub Issue for your task** — Before starting any coding, create a new issue in your GitHub repository describing the feature or bug you want Claude to work on. This issue serves as the "ticket" that Claude will read and plan against.

    You should see a new GitHub Issue created with a clear title and description of the task.

6.  **Initiate the workflow with Claude Code** — Open your Claude Code CLI or web interface. Navigate to your project directory (if using CLI) or ensure Claude has access to your repository. Then, type `/wizard` to activate the skill and provide the link to your GitHub Issue.

    ```
    /wizard [link to your GitHub Issue]
    ```

    Claude Code should respond by reading your `CLAUDE.md` and the GitHub Issue, then present a structured to-do list and complexity assessment, as defined in Phase 1 of your `wizard.md` file. It will then wait for your confirmation or further instructions to proceed (presumably to Phase 2, which is not detailed in the source).

## Notes & variations
*   **Free-tier alternatives:** While Claude Code is the primary tool here, the concept of guiding an AI with a structured prompt can be applied to other large language models (LLMs) like ChatGPT or Gemini, though the `/wizard` skill activation might differ. For CI, alternatives to GitHub Actions include GitLab CI/CD or Jenkins.
*   **Common mistake:** A common pitfall is not having a clear, comprehensive `CLAUDE.md` file or poorly defined GitHub Issues. Claude's output quality is directly proportional to the clarity and detail of its inputs and guiding instructions. Ensure your `CLAUDE.md` is up-to-date and your issues are specific.
*   **Tip for better results:** Continuously refine your `CLAUDE.md` and the `wizard.md` skill based on the specific needs of your projects and the quality of Claude's output. Break down complex tasks into smaller, manageable GitHub Issues to help Claude focus and produce more accurate results. Experiment with adding more phases to your `wizard.md` (if you can infer or design them) to guide Claude through the entire development lifecycle.
