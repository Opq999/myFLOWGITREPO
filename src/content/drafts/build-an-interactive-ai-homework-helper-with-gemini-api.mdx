## What you'll get
You will build an interactive AI agent that can gamify children's homework by visually inspecting their work, testing their knowledge, and dispensing physical rewards. This approach uses AI's multimodal capabilities to create an engaging and automated reward system, ideal for motivating young learners.

## Tools you need
*   **Gemini API** (freemium): To power the AI agent's reasoning, multimodal understanding (image analysis), and tool interaction.
*   **Google Agent Development Kit (ADK)** (free): A framework for building AI agents, managing conversations, and integrating tools.
*   **FastAPI** (free): A modern, fast web framework for building APIs with Python, used here as the foundation for the agent.
*   **FT232H USB to GPIO converter** (paid): A chip that allows your computer to control physical hardware like motors or locks via USB.
*   **Raspberry Pi** (paid): A small, affordable computer used here to manage hardware like an LED matrix, communicating with the main agent.

## Steps
1.  **Set up your development environment**: Install necessary Python libraries and configure your Google Cloud project. The author uses `agent-starter-pack` which includes FastAPI and frontend integration. You will need to set environment variables for your Google Cloud project ID and location. The author also sets `GOOGLE_GENAI_USE_VERTEXAI` to "True" and an `AGENT_LANGUAGE` variable, defaulting to English.
    *   **What to do**: Follow the project's setup instructions to install dependencies and configure your Google Cloud credentials. Set the `GOOGLE_CLOUD_PROJECT`, `GOOGLE_CLOUD_LOCATION`, and `GOOGLE_GENAI_USE_VERTEXAI` environment variables. Optionally, set `AGENT_LANGUAGE` to "pl" for Polish.
    *   **What you should see**: Your environment is ready for agent development, with all required packages installed and configurations loaded.

2.  **Initialize the Gemini Agent Platform**: Configure the agent to use Vertex AI, Google's managed machine learning platform. This involves initializing the Vertex AI SDK with your project details.
    *   **What to do**: Ensure your code includes `vertexai.init(project=project_id, location=location)` where `project_id` and `location` are loaded from your environment variables.
    *   **What you should see**: The Vertex AI SDK is successfully initialized, allowing your agent to access Gemini models.

3.  **Define the root agent**: Create the main agent instance using the ADK. This agent will orchestrate the interaction with the child, process homework, and decide on rewards.
    *   **What to do**: In your `agent/app/agent.py` file, instantiate the agent. The author's code shows `root_agent = A...` which is the start of the agent definition. You will need to define the agent's persona, its capabilities, and how it interacts with tools.
    *   **What you should see**: A Python object representing your AI agent is created and ready to be configured with specific tasks and tools.

4.  **Integrate multimodal homework validation**: Configure the agent to accept image uploads of homework. The Gemini API will analyze these images to verify completion and correctness.
    *   **What to do**: Define a tool or function within your agent that accepts an image input. This function will pass the image to the Gemini API for analysis. The author mentions "visual inspection of their workbooks via uploaded images".
    *   **What you should see**: The agent can process image data and use Gemini's multimodal capabilities to understand the content of the homework.

5.  **Implement verbal comprehension checks**: Add functionality for the agent to ask children questions about their reading material to test comprehension.
    *   **What to do**: Create a tool or function that generates comprehension questions based on provided text (e.g., the reading material). The agent will then present these questions to the child.
    *   **What you should see**: The agent can dynamically generate questions and engage the child in a verbal quiz.

6.  **Connect to physical reward hardware**: Set up the agent to trigger physical actions, such as unlocking a drawer containing sweets, upon successful homework completion.
    *   **What to do**: Define a tool that interfaces with the FT232H USB to GPIO converter. This tool will send commands to unlock specific drawers. The author's system uses "Magnetic drawers controlled via FT232H USB to GPIO converter".
    *   **What you should see**: When the agent determines the homework is complete, it can send a signal to the FT232H to activate the reward mechanism.

7.  **Manage state across conversation turns**: Ensure the agent remembers the context of the conversation, including previous interactions and the child's progress.
    *   **What to do**: Utilize the ADK's state management features to store and retrieve information across multiple turns of the conversation. This is crucial for tracking homework progress and reward dispensing.
    *   **What you should see**: The agent maintains a coherent conversation flow, remembering what has been discussed and completed.

8.  **Run the agent locally**: Develop and test the agent on your local machine to enable direct access to the physical hardware.
    *   **What to do**: Use the provided FastAPI setup to run the agent locally. Ensure your FT232H converter and Raspberry Pi (if used for the LED matrix) are connected and recognized by your system.
    *   **What you should see**: The agent is running, accessible via a local web interface, and can interact with the connected hardware.

## Notes & variations
*   **Free-tier alternatives**: While the Gemini API has a freemium model, ensure you monitor usage to stay within free limits. For physical hardware, alternatives to FT232H might exist depending on your specific reward mechanism, but direct GPIO control often requires specific hardware.
*   **Common mistake**: Forgetting to properly configure Google Cloud credentials or environment variables can prevent the agent from initializing or accessing the Gemini API. Double-check all paths and keys.
*   **Tip to get better results**: Clearly define the agent's persona and interaction style. Making the agent sound encouraging and fun will significantly improve a child's engagement with the homework gamification system.
