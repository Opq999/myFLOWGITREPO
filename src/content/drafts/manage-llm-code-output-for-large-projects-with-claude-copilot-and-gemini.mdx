## What you'll get

You will learn a method to manage large code outputs from AI models, preventing common issues like truncations and hallucinations. This approach is crucial when working with extensive codebases as a non-developer, ensuring that AI-generated code integrates smoothly without breaking existing logic.

## Tools you need

*   **Claude** (freemium) - An AI assistant for generating and refining code.
*   **Copilot** (paid) - An AI pair programmer that suggests code and entire functions in real-time.
*   **Gemini** (freemium) - Google's AI model for various tasks, including code generation.

## Steps

1.  **Identify the code block to modify** — Before asking the LLM to make changes, pinpoint the exact section of your code that needs updating. This could be a specific function, a class, or a block of HTML.
2.  **Formulate a precise instruction** — Tell the LLM to act as a patch tool. Instead of asking for a full file rewrite, instruct it to find a specific block of text and replace it with new code. The author's method was to ask the LLM to output "Find this exact block" and "Replace with this exact block."
3.  **Provide the existing code block** — Paste the exact code block you identified in step 1 into your prompt. This gives the LLM the precise context it needs.
4.  **Provide the new code** — Clearly present the new code you want the LLM to insert. Ensure it's well-formatted and distinct from the existing block.
5.  **Construct the patch prompt** — Combine the instructions and code snippets into a single prompt. For example:

    ```
    Find this exact block:
    [Paste the exact existing code block here]
    Replace with this exact block:
    [Paste the new code block here]
    ```

    You should receive a response from the LLM that clearly indicates it has understood the find-and-replace operation, and it will provide the modified code.
6.  **Integrate the changes** — Carefully copy the LLM's output and replace the original code block in your project with the new one. Always test thoroughly after making changes.

## Notes & variations

*   **Free tier alternative:** While Copilot is paid, you can achieve similar results using the free tiers of Claude or Gemini by carefully crafting your prompts to request specific code modifications.
*   **Common pitfall:** Avoid asking LLMs to rewrite entire files or large sections of code at once. This increases the risk of unexpected errors, truncations, or subtle changes (like incorrect quote replacements) that can break your project.
*   **Tip for better results:** Always keep a version control system (like Git) active. This allows you to easily revert to a previous working state if an AI-generated code change introduces bugs.
