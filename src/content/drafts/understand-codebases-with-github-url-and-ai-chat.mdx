## What you'll get
You will be able to paste a public GitHub repository URL and ask questions about its code in plain English. The tool will generate a live dependency graph, show file contents, display the full file tree, and provide explanations via a chat interface, all updating in real-time. This approach helps you quickly grasp the structure and functionality of unfamiliar codebases without getting lost in details.

## Tools you need
*   **CopilotKit** (freemium) - Provides AI agent UI components and state synchronization for chat interfaces.
*   **Zenflow** (paid, but workflow can be replicated) - An AI development tool used for planning and orchestrating software builds; its core functionality can be understood and adapted.
*   **GitHub API** (free) - Allows programmatic access to GitHub repositories, used here to fetch file trees and content.
*   **React Flow** (free) - A library for building interactive node-based UIs, used here for visualizing dependency graphs.
*   **Ollama** (free) - Enables running large language models (LLMs) locally on your machine for free inference.
*   **Next.js** (free) - A React framework for building web applications, used for the frontend and API routes.
*   **Octokit** (free) - A JavaScript client for the GitHub API.
*   **Zustand** (free) - A small, fast, and scalable bearbones state-management solution for React.
*   **Tailwind CSS** (free) - A utility-first CSS framework for rapid UI development.
*   **OpenAI** (paid) / **Ollama** (free) - LLM backend for processing natural language queries.

## Steps
1.  **Set up the Codebase Navigator environment** - The original source describes setting up a Next.js project with CopilotKit, Zenflow, React Flow, Octokit, Zustand, and Tailwind CSS. For a beginner, this involves cloning a starter repository and installing dependencies. The author doesn't provide exact commands, but a typical setup would involve:
    *   Cloning the project repository (if a starter is provided).
    *   Running `npm install` or `yarn install` in the project directory to install all necessary packages.
    *   Configuring environment variables, particularly for any API keys if not using Ollama locally.
    You should see a confirmation that all packages have been installed.
2.  **Configure the LLM backend** - If you want to run this workflow for free, you need to set up Ollama locally. Download and install Ollama from its official website. Once installed, you can pull a compatible LLM model by running a command like `ollama pull llama3` in your terminal. The author mentions that the CopilotKit runtime can be configured to use any OpenAI-compatible backend, including Ollama.
    You should see output indicating the model is downloading or has been successfully pulled.
3.  **Launch the application** - Start the Next.js development server. Typically, this is done by running `npm run dev` or `yarn dev` in your project's terminal.
    You should see output indicating the development server is running, usually on `http://localhost:3000`.
4.  **Input a GitHub URL** - Open the application in your web browser. Locate the input field where you can paste a GitHub repository URL. Enter the URL of any public GitHub repository you wish to explore.
    The application should accept the URL and begin processing it.
5.  **Ask a question about the codebase** - After the repository is loaded and the dependency graph is generated, you will see a chat interface. Type your question into the chat input box. For example, to understand how user authentication works, you might type:

    ```
    how does auth work?
    ```

    You should see the chat interface respond with an explanation, and potentially highlight relevant files or parts of the dependency graph.
6.  **Explore the dependency graph** - Observe the generated graph canvas. This visual representation shows how different files in the repository import or depend on each other. As you ask questions or navigate, the graph may update to highlight relevant connections.
    The graph should display nodes representing files and edges representing import statements.
7.  **View file content** - Use the repo explorer panel to browse the file tree. Clicking on a file in the tree should load its content into the code viewer panel, with relevant lines highlighted based on your questions or the current context.
    The code viewer should display the source code of the selected file.

## Notes & variations
*   **Free Tier Focus**: The primary way to use this workflow for free is by leveraging Ollama for local LLM inference, avoiding costly API calls. Ensure your machine meets Ollama's system requirements.
*   **Common Pitfall**: If the dependency graph appears incomplete or incorrect, double-check that the GitHub repository is public and that the tool has successfully fetched all necessary file paths. Sometimes, private repositories or complex build configurations can cause issues.
*   **Better Results**: For more precise answers, provide context in your questions. Instead of just "how does X work?", try "In the context of the user authentication module, how does the login process work?" This helps the AI focus its analysis.
