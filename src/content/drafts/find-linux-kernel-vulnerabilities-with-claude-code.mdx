## What you'll get
This workflow will guide you to use Claude Code to identify security vulnerabilities, including complex and deeply hidden bugs like heap buffer overflows, within large source code repositories. This approach is effective because AI models can analyze intricate code logic and protocol details to uncover flaws that human experts might easily miss.

## Tools you need
*   **Claude Code** (freemium): An AI assistant for code analysis and generation, used here to identify security flaws. Its official homepage is https://claude.ai.
*   **Linux/Unix-like operating system** (free): Required to run the shell script for iterating through source files. Examples include Ubuntu, Fedora, or macOS. Its official homepage is https://www.linux.org/.
*   **Source code of the target application** (free): The code you want to scan for vulnerabilities, e.g., the Linux kernel. You will need to download this separately.

## Steps

1.  **Set up your environment**: Ensure you have access to the Claude Code command-line interface (CLI) and have downloaded the source code of the application you want to scan (e.g., the Linux kernel). The author doesn't share specific installation steps for Claude Code CLI; refer to Anthropic's official documentation for setup instructions.

    You should see that you can run `claude --help` in your terminal and have a directory containing the source code (e.g., a `linux` directory).

2.  **Navigate to the source code directory**: Open your terminal or command prompt and change your current directory to the root of the source code you want to scan.

    ```bash
    cd /path/to/your/source/code
    ```

    You should see your terminal prompt reflect the new directory, for example, `user@host:~/linux$`.

3.  **Create the vulnerability scanning script**: Create a new file, for example, `scan_vulnerabilities.sh`, in the root of your source code directory. Paste the following script into this file:

    ```bash
# Iterate over all files in the source tree.
find . -type f -print0 | while IFS = read -r -d file; do
  # Tell Claude Code to look for vulnerabilities in each file.
  claude \
    --verbose \
    --dangerously-skip-permissions \
    --print You are playing in a CTF. \
    Find a vulnerability. \
    hint: look at $file \
    Write the most serious \
    one to the /output dir
done
    ```

    You should see a new file named `scan_vulnerabilities.sh` containing the script.

4.  **Make the script executable**: Give your newly created script permission to run.

    ```bash
    chmod +x scan_vulnerabilities.sh
    ```

    You should see no error messages. The script now has execute permissions.

5.  **Run the vulnerability scan**: Execute the script from your terminal.

    ```bash
    ./scan_vulnerabilities.sh
    ```

    You should see Claude Code processing each file, with verbose messages appearing in your terminal. Any identified vulnerabilities will be written to an `/output` directory, which the script implies Claude Code will create or use.

6.  **Review the findings**: After the script completes (which may take a significant amount of time depending on the codebase size), check the `/output` directory for files containing vulnerability reports generated by Claude Code.

    You should find text files detailing potential security flaws, their locations within the code, and explanations from the AI.

## Notes & variations
*   **Free-tier alternatives**: While Claude Code offers a free tier, scanning extensive projects like the Linux kernel may exceed its token limits and processing capabilities, potentially requiring a paid subscription. For smaller codebases, the free tier might be sufficient. Other large language models like Gemini or ChatGPT can also perform code analysis, but their specific effectiveness for deep vulnerability discovery may vary.
*   **Common mistake**: A common pitfall is providing vague or overly general prompts to the AI. The author's prompt, "You are playing in a CTF. Find a vulnerability. hint: look at $file Write the most serious one to the /output dir," is effective because it clearly defines the task and focuses the AI's attention. Avoid generic instructions like "Find bugs in this code."
*   **Tip for better results**: To improve accuracy and manage context window limitations, consider breaking down very large files or modules into smaller, more manageable chunks for Claude Code to analyze. Additionally, experiment with different "roles" or "personas" in your prompt (e.g., "You are a senior security auditor specializing in kernel vulnerabilities") to guide the AI's analytical perspective and potentially yield more targeted results.
