## What you'll get
You will learn how to set up your workspace to monitor multiple AI coding agents concurrently, using VR to view several VSCode windows at once. This approach helps manage the complexity of multi-agent AI coding by keeping critical information like code and diffs within your field of view.

## Tools you need
*   **VR Headset** (paid) - To create an immersive environment where multiple virtual monitors can be displayed.
*   **VSCode** (free) - A popular, free source-code editor used for writing and managing code.

## Steps
1.  **Prepare your AI coding environment** - Ensure you have multiple AI coding agents ready to be launched on different tasks or repositories. The author mentions using agents for tasks like editing, thinking, testing, or building code.
2.  **Launch AI agents and tasks** - Start 4-5 AI agents simultaneously, assigning each a different task in a separate repository. The goal is to have multiple coding processes running concurrently.
3.  **Open VSCode for each agent** - For each AI agent, open its corresponding VSCode window. You need to see the code and the diffs the agent is producing, not just the terminal output.
4.  **Arrange VSCode windows in VR** - Put on your VR headset and arrange the VSCode windows within your virtual workspace. The author's goal is to have all 4-5 agent VSCode windows visible within a single field of view.
5.  **Split VSCode windows (Optional)** - The author notes that they often split each VSCode window into two parts to view different aspects of the code or diffs simultaneously. This is a manual and flexible arrangement.
6.  **Monitor code and diffs** - Actively observe the code changes and differences (diffs) presented in each VSCode window. This is crucial for catching potential errors or unintended changes made by the AI agents.

## Original source
This workflow is based on insights shared by author samchon on the DEV Community platform. The author discusses how the rise of AI coding necessitates new ways to monitor multiple agents, leading to the adoption of VR for enhanced workspace management.

## Notes & variations
*   **Free-tier alternative:** While VR is a paid tool, the core principle of managing multiple windows can be partially replicated on a standard computer by using window management software or simply tiling multiple VSCode windows on your desktop. However, this will not provide the same immersive, single-field-of-view benefit.
*   **Common pitfall:** Trying to actively read and process all 5 diffs at the exact same time. The author clarifies that you look at one place at a time, but having them all visible in your peripheral view is key for quick checks and context switching.
*   **Tip for better results:** Regularly review the diffs generated by the AI agents, especially in complex codebases. Do not solely rely on the agent's summary, as subtle but dangerous changes can be hidden within the code modifications.
