## What you'll get
You will learn to transform your interaction with Claude Code from a simple prompt-and-wait chatbot into a powerful, programmable agent for software development. This workflow will enable Claude to verify its own work, leading to a 2-3x quality improvement in code generation and refactoring.
This approach works by establishing guardrails and feedback loops, allowing Claude to iterate and refine code until it actually runs, significantly reducing manual intervention and speeding up development cycles.

## Tools you need
*   **Claude Code** (freemium): An AI coding assistant that runs in your terminal, used here as an autonomous agent for code generation and refactoring.
*   **Obsidian** (freemium): A knowledge base and note-taking tool, used here for advanced workflow management with Claude Code (though specific steps for Obsidian are not detailed in the source, it's mentioned as part of an advanced workflow).

## Steps
1.  **Shift your mindset to an autonomous agent**: Treat Claude Code as an engineer you delegate to, not a pair programmer. Give Claude a way to verify its own work, enabling self-correction and iteration until code runs correctly.
    *   *You'll see*: A fundamental shift in interaction, moving from simple prompts to delegating tasks.
2.  **Plan complex changes with Plan Mode**: For code changes touching multiple files, open Claude Code and press `Shift+Tab` twice to enter "plan mode." Let Claude read files, trace flows, and map the data model.
    *   *You'll see*: Claude generates a detailed, read-only plan.
3.  **Refine Claude's plan in your editor**: After Claude presents its plan, press `Ctrl+G`. This opens the plan as a text file in your editor. Review and modify it to align with your requirements.
    *   *You'll see*: The plan appears in your editor, ready for your edits.
4.  **Provide exact context with direct references**: Instead of describing files or errors, directly reference them. Use `@` followed by the file path for code. Pipe error logs directly for debugging.
    *   *Example prompt for file reference*:

    ```
    Refactor the authentication logic in this file to improve security: @src/auth/login.py
    ```

    *   *Example command for error context*:

    ```bash
    cat error.log | claude "Analyze this error log and suggest a fix for the issue."
    ```

    *   *You'll see*: Claude receives precise context, leading to more accurate suggestions.
5.  **Delegate tasks with clear briefs**: Write a crisp, upfront brief for Claude, clearly stating the objective and expected outcome. Avoid line-by-line guidance.
    *   *Example prompt*:

    ```
    Implement a new API endpoint for user profile updates. It should accept POST requests to /api/users/{id} with JSON data containing 'name' and 'email' fields. Ensure proper input validation and update the database.
    ```

    *   *You'll see*: Claude performs best with autonomy based on a well-defined brief.
6.  **Teach Claude from its mistakes**: If Claude makes a mistake, provide feedback and instruct it to update its internal rules.
    *   *Example prompt (after a mistake)*:

    ```
    The previous implementation did not handle edge cases for empty input correctly. Please revise the function to include robust validation. Update CLAUDE.md so you don't repeat this error in the future.
    ```

    *   *You'll see*: Claude learns from failures, improving future performance via `CLAUDE.md` updates.
7.  **Understand Claude's configuration files**: Explore `.claude/` (project scope) and `~/.claude/` (global scope). `CLAUDE.md` holds general instructions, `CLAUDE.local.md` is for private notes (gitignore it), and `settings.json` manages permissions/environment variables.
    *   *You'll see*: A clear mental model of Claude's persistent memory and configurations.
8.  **Review plans with a "second Claude" (advanced)**: For critical changes, open a new Claude Code session. Provide it with the plan from your first session and ask it to review as an independent staff engineer, without prior context.
    *   *Example prompt for the second Claude*:

    ```
    I have a proposed plan for a significant code refactor. Please review this plan as an experienced staff engineer, identify any potential gaps, risks, or areas for improvement.
    [Paste the plan generated by the first Claude session here]
    ```

    *   *You'll see*: An unbiased, critical assessment of the plan, helping catch potential issues.

## Notes & variations
*   **Free-tier alternatives**: While Claude Code itself offers a free tier, the principles of agentic behavior and self-correction can be applied to other advanced large language models if they support similar interaction patterns, such as file referencing and persistent custom instructions.
*   **Common mistake**: A common pitfall is treating Claude Code as a simple chatbot or autocomplete tool, rather than an autonomous agent capable of self-correction. This leads to missed opportunities for quality improvement and efficiency.
*   **Tip for better results**: Consistently use the `Update CLAUDE.md` habit. The author notes this single habit compounds harder than anything else, allowing Claude to continuously learn from its own failures and improve its performance over time.
