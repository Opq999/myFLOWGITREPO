## What this is, in plain English
This workflow describes an advanced setup for running multiple AI coding agents on a remote cloud server, which you can control entirely from your mobile phone. The core idea is to offload heavy computing tasks to a powerful server (a VM, or Virtual Machine) and interact with it using a mobile terminal app that stays connected even if your phone loses signal. When an AI agent needs your input, you get a push notification on your phone, allowing you to respond and continue working without needing a laptop or desktop.

This is an advanced concept, not a simple copy-paste recipe. It requires significant technical skill and comfort with cloud infrastructure, networking, command-line tools, and scripting. The exact steps involve setting up a virtual server, configuring secure network access, installing various command-line utilities, and writing custom scripts to manage the server and integrate push notifications.

Because the setup involves many moving parts (a cloud VM, a VPN, a mobile terminal, a session manager, an AI API, and custom scripting), there isn't a single set of instructions that a beginner can follow. The author's specific configuration details live across various configuration files and scripts that are not fully provided here.

## How it actually works
This setup involves several advanced components working together. Here's a high-level overview of the steps an advanced user would take:

1.  **Set up a cloud Virtual Machine (VM)**: You would provision a server from a cloud provider like Vultr. This VM will host your AI agents and development environment. You'd configure it for secure access, typically by blocking public SSH and routing all connections through a private network like Tailscale.
2.  **Install mobile terminal and tools**: On your mobile phone, you'd install Termius to connect to the VM. On the VM itself, you'd install `mosh` for resilient connections, `tmux` for session persistence, and command-line tools like `jq` and `curl`.
3.  **Configure Claude Code**: You would set up Claude Code on the VM. The author configures a `PreToolUse` hook in Claude Code's settings. This hook is a special instruction that tells Claude Code to run a custom script whenever it needs input from the user (e.g., when it asks a question).
4.  **Implement push notifications**: The custom script triggered by Claude Code's hook extracts the question the AI agent needs answered. It then uses `curl` to send this question as a message to a push notification service's webhook (like Poke), which then delivers the notification to your phone.
5.  **Manage VM lifecycle**: To control costs, the author uses scripts or an iOS Shortcut that calls the Vultr API to start and stop the VM only when needed. This ensures you only pay for the server when you are actively using it.
6.  **Enable parallel development**: The author uses `tmux` to create multiple independent terminal windows on the VM, each running a separate Claude Code agent. They also leverage Git worktrees, a Git feature that allows multiple working directories for the same repository, to work on different features or branches simultaneously.

Exact commands and configuration values for firewall rules, API keys, and specific script contents are not provided in the source and would need to be developed by the user based on their specific setup and security requirements.

## Notes & variations
*   **Do you even need this?** For most users, this complex setup is overkill. If you only need occasional AI coding assistance, simpler alternatives include using web-based AI chat applications (like claude.ai or chatgpt.com) directly on your phone or laptop, or running local large language models (LLMs) on a powerful computer. This workflow is specifically for advanced users who require continuous, parallel AI agent operation and deep integration into a mobile-first development routine.
*   **Free-tier limits**: While Tailscale and Termius offer useful free tiers, the core components of this workflow (Vultr for the VM, Claude Code for the AI agent, and Poke for notifications) are paid services. Be mindful of hourly VM costs and API usage fees, as these can add up quickly if not managed carefully.
*   **Common pitfall**: A major challenge is managing security and cost. Forgetting to stop your VM can lead to unexpected charges. Additionally, improperly securing your VM or exposing API keys can create significant security risks. Always follow best practices for cloud security, including strong passwords, multi-factor authentication, and strict firewall rules.
